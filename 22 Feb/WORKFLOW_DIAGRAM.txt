KEY POINTS
═══════════════════════════════════════════════════════════════════════════════

1. STATUS TRANSITIONS:
   SUBMITTED → PENDING_SA_APPROVAL → APPROVED → (Work begins)

2. AUTO-POPULATED FIELDS:
   ✓ requester (from JWT principal at creation)
   ✓ organizationDepartment (from user profile)
   ✓ admin name and timestamp (from JWT at review)
   ✓ super admin name and timestamp (from JWT at review)

3. APPROVAL GATES:
   • Admin MUST set requiredDate before approval
   • Super Admin MUST provide quotation before approval
   • Each stage has clear approval/rejection paths

4. AUDIT TRAIL:
   • Every reviewer is recorded with timestamp
   • All remarks are preserved for transparency
   • Status changes are immutable

5. ROLE RESTRICTIONS:
   • Only Admin can review SUBMITTED requests
   • Only Super Admin can review PENDING_SA_APPROVAL requests
   • Requesters can only see their own requests

6. QUOTATION DETAILS:
   • Itemized breakdown provided
   • Total cost clearly stated
   • Description helps requester understand charges


ERROR HANDLING EXAMPLES
═══════════════════════════════════════════════════════════════════════════════

1. Admin tries to review PENDING_SA_APPROVAL request:
   ❌ 409 Conflict - Request not in SUBMITTED status

2. Super Admin tries to review SUBMITTED request:
   ❌ 409 Conflict - Request not in PENDING_SA_APPROVAL status

3. Requester tries to view others' requests:
   ❌ 404 Not Found - Access denied

4. Missing required fields in admin review:
   ❌ 400 Bad Request - requiredDate, adminRemarks required

5. Negative quotation amount:
   ❌ 400 Bad Request - Quotation amount must be positive
